# Community Trademark Use Policies

> Last updated: 2026-03-15

## Summary

Open source projects need trademark use policies that balance brand protection with community use. These policies define how third parties can reference the project name, under what conditions they can use the logo, and what restrictions apply to modified or derivative versions.

The Model Trademark Guidelines project (modeltrademarkguidelines.org) provides templates used by many open source foundations.

## Common Policy Elements

### Permitted Uses (Typically)
- Stating compatibility ("Works with Linux")
- Factual descriptions ("Built on Apache Kafka")
- Community discussion and non-commercial reference
- Conference/meetup names (with appropriate disclaimer)

### Restricted Uses (Typically)
- Modified software distributed under the original name
- Commercial products/services using the project name without permission
- Domain names incorporating the trademark
- Implying endorsement or affiliation without authorization

### Modified Version Naming
- Most policies require that modified versions use a different name
- Exception: if modifications are limited and comply with compatibility requirements
- Example: Kubernetes conformance certification allows use of the "Kubernetes" name for certified distributions

## Practical Guidance

1. **Use the Model Trademark Guidelines as a starting point.** Available at modeltrademarkguidelines.org — provides a well-drafted, community-tested template.

2. **Be clear about what requires permission.** Ambiguous trademark policies create more problems than no policy at all.

3. **Balance protection with community growth.** Overly restrictive trademark policies discourage community adoption and contribution.
