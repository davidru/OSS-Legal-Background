# Glossary of Open Source Legal Terms

> Last updated: 2026-03-15

## A

**AGPL (Affero General Public License):** A strong copyleft license that extends GPL's source code obligations to network use (SaaS).

**AFC Test (Abstraction-Filtration-Comparison):** The dominant judicial test for non-literal software copyright infringement, from Computer Associates v. Altai (2d Cir. 1992).

**Apache License 2.0:** A permissive open source license with an express patent grant and patent retaliation clause.

**Attribution:** The requirement to preserve copyright notices and license text when distributing open source software.

## B

**BDFL (Benevolent Dictator for Life):** A governance model where a single person has final decision authority.

**BSD License:** A family of permissive open source licenses originating from the University of California, Berkeley.

**BSL (Business Source License):** A source-available license where usage restrictions expire after a set period, at which point the code becomes open source.

## C

**CLA (Contributor License Agreement):** A legal agreement between a contributor and a project granting specific IP rights.

**Copyleft:** A licensing approach requiring derivative works to be distributed under the same license terms.

**Corresponding Source:** GPLv3's term for the complete source code needed to build a binary, including build scripts and installation information.

**COTS (Commercial Off-The-Shelf):** Software available for purchase/use without custom development. Open source qualifies as COTS for US government procurement.

## D

**DCO (Developer Certificate of Origin):** A lightweight attestation mechanism where contributors certify they have the right to submit their contribution.

**Derivative Work:** Under US copyright law (17 U.S.C. § 101), a work based upon a preexisting work. The scope of this term in the GPL context is heavily contested.

**Dual Licensing:** A business model offering software under both an open source license and a commercial/proprietary license.

## E

**EAR (Export Administration Regulations):** US export control regulations. Publicly available open source is generally exempt.

**EPL (Eclipse Public License):** A weak copyleft license with module-level copyleft scope.

## F

**Fair Use:** A US copyright defense (17 U.S.C. § 107) permitting certain uses of copyrighted works without permission.

**FRAND (Fair, Reasonable, and Non-Discriminatory):** Patent licensing terms commonly required by standards organizations.

**FOSS (Free and Open Source Software):** Software that meets both the Free Software Definition and the Open Source Definition.

## G

**GPL (General Public License):** The most widely used copyleft license, published by the Free Software Foundation. Current version is GPLv3 (2007).

## I

**Implied Patent License:** A patent license inferred from conduct (e.g., distributing software), as opposed to an express written grant.

## L

**LGPL (Lesser General Public License):** A weak copyleft license designed for libraries, permitting linking from proprietary code.

**Linking (Static/Dynamic):** Methods of combining software components. The legal significance for copyleft scope is heavily debated.

## M

**MPL (Mozilla Public License):** A weak copyleft license with file-level copyleft scope.

**Merger Doctrine:** The principle that when an idea can be expressed in only limited ways, the expression merges with the idea and is uncopyrightable.

## N

**Necessary Claims:** Patent claims that are necessarily infringed by implementing a specification or using software as designed.

## O

**OIN (Open Invention Network):** A patent cross-license community for the Linux ecosystem.

**Open Source Definition:** The Open Source Initiative's set of criteria that a license must meet to be considered "open source."

**OSAID (Open Source AI Definition):** OSI's definition of what constitutes "open source" for AI systems.

## P

**Patent Retaliation:** A license clause terminating patent (and sometimes all) rights if the licensee sues for patent infringement.

**Patent Pledge:** A unilateral commitment by a patent holder not to assert patents against certain activities.

**Permissive License:** An open source license with minimal conditions (typically attribution only), not requiring derivative works to use the same license.

## R

**RF (Royalty-Free):** Patent licensing terms that do not require payment of royalties.

## S

**SBOM (Software Bill of Materials):** A machine-readable inventory of all software components in a product.

**Scènes à Faire:** Elements of a work dictated by external factors (hardware, conventions, standards) that are not copyrightable.

**SEP (Standard-Essential Patent):** A patent that is necessarily infringed by implementing a standard.

**Source-Available:** Software whose source code is accessible but whose license imposes restrictions incompatible with the Open Source Definition.

**SPDX (Software Package Data Exchange):** A standard for communicating software component and license information.

**SSO (Structure, Sequence, and Organization):** Non-literal elements of software that may (or may not) be copyrightable.

## W

**Work-for-Hire:** Under US copyright law (17 U.S.C. § 101), works created by employees within the scope of employment, owned by the employer.

**Written Offer:** A GPL mechanism for satisfying source code obligations by providing a written promise to deliver source code upon request.
