# Historical Timeline of Open Source Software Law

Key milestones in the legal development of free and open source software.

> **Status:** Initial draft — contributions welcome. Dates reflect the year of the event; see linked sources for precise dates.

## 1980s — Foundations

| Year | Event | Significance |
|---|---|---|
| 1980 | US Copyright Act amended (CONTU recommendations) | Software explicitly protected by copyright |
| 1983 | Richard Stallman announces GNU Project | Birth of the free software movement |
| 1985 | Free Software Foundation (FSF) incorporated | Institutional home for free software advocacy |
| 1989 | GPL v1 released | First copyleft license |
| 1989 | *Lotus v. Paperback Software* | Menu command hierarchy copyrightable (later weakened) |

## 1990s — The Open Source Emergence

| Year | Event | Significance |
|---|---|---|
| 1991 | GPL v2 released | Became the dominant copyleft license for decades |
| 1991 | Linux kernel released under GPL v2 | Most consequential GPL-licensed project |
| 1992 | *Computer Associates v. Altai* | Abstraction-Filtration-Comparison test for software copyright |
| 1995 | Apache Group forms, releases Apache License | Foundation of permissive licensing tradition |
| 1997 | *The Cathedral and the Bazaar* (Eric Raymond) | Intellectual framework for open source development |
| 1998 | "Open Source" term coined; OSI founded | Open Source Definition published; movement splits from "free software" branding |
| 1998 | Netscape releases Mozilla source code | First major commercial-to-open-source conversion |
| 1999 | Apache Software Foundation incorporated | Model for foundation-based OSS governance |

## 2000s — Enforcement & Enterprise Adoption

| Year | Event | Significance |
|---|---|---|
| 2003 | SCO v. IBM filed | High-profile challenge to Linux IP; eventually failed |
| 2004 | gpl-violations.org begins enforcement (Harald Welte) | First systematic GPL enforcement in Europe |
| 2004 | *Munich Landgericht* — Welte v. Sitecom | First court injunction enforcing the GPL |
| 2006 | GPLv3 drafting process begins (public) | Unprecedented transparent license drafting process |
| 2007 | GPL v3 released | Added patent, anti-tivoization, and compatibility provisions |
| 2007 | *Jacobsen v. Katzer* (Fed. Cir.) | Artistic License conditions enforceable as copyright conditions (not mere covenants) |
| 2008 | *Welte v. Skype* (Munich) | GPL enforcement upheld against consumer electronics company |
| 2008 | GitHub launches | Transformed OSS collaboration and distribution |
| 2009 | Oracle acquires Sun Microsystems | Began era of corporate stewardship questions (Java, MySQL, OpenOffice) |

## 2010s — Maturity & New Challenges

| Year | Event | Significance |
|---|---|---|
| 2010 | *Oracle v. Google* filed | API copyrightability — decade-long litigation begins |
| 2012 | *Oracle v. Google* (N.D. Cal.) | APIs not copyrightable (Alsup, J.) — reversed on appeal |
| 2014 | *Oracle v. Google* (Fed. Cir.) | API structure copyrightable — circuit split implications |
| 2014 | *Versata v. Ameriprise* (Texas) | GPL enforceability as breach of contract |
| 2015 | *Patrick McHardy* GPL enforcement campaigns begin | Controversial aggressive enforcement in Europe |
| 2016 | *Artifex v. Hancom* (N.D. Cal.) | GPL breach survives motion to dismiss as both copyright and contract claim |
| 2017 | *SFC v. Vizio* filed | First GPL enforcement as third-party beneficiary (consumer standing) |
| 2018 | SSPL published by MongoDB | Begins wave of "source-available" relicensing |
| 2018 | EU Copyright Directive adopted | New obligations affecting code hosting platforms |
| 2018 | OpenChain becomes ISO 5230 (process begins) | Standardization of OSS compliance programs |
| 2019 | *Hancom v. Artifex* settlement | GPL enforceability confirmed through litigation outcome |

## 2020s — AI, Security & Regulation

| Year | Event | Significance |
|---|---|---|
| 2021 | *Google v. Oracle* (US Supreme Court) | Fair use defense for API reimplementation — landmark ruling |
| 2021 | Log4Shell vulnerability | OSS supply chain security becomes mainstream policy concern |
| 2021 | GitHub Copilot launched | AI code generation trained on OSS — licensing questions emerge |
| 2022 | EU Cyber Resilience Act proposed | Regulatory obligations potentially applicable to OSS maintainers |
| 2022 | *SFC v. Vizio* (9th Cir.) | State court contract claims for GPL enforcement can proceed |
| 2022 | US Executive Order on Software Supply Chain | SBOM requirements for government procurement |
| 2023 | CRA amended with open source steward provisions | EU regulation acknowledges OSS ecosystem role |
| 2023 | HashiCorp relicenses to BSL | Terraform fork (OpenTofu) highlights relicensing tensions |
| 2024 | Redis relicenses to dual SSPL/RSALv2 | Continued trend of cloud-related relicensing |
| 2024 | OSI releases Open Source AI Definition (OSAID) | First attempt to define "open source" for AI models |
| 2024 | EU AI Act enters into force | Intersections with open source AI model distribution |
